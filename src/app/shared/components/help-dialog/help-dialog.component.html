Help

Creator Role
Project creator has the highest level of access to the project. Project creator can add or remove the members or change their roles in the project. Creator can add, update and delete the tasks and attachments. Project can only be deleted by the creator.
Admin Role
Project's admin can update the project, add or remove members. They can update the project members roles except their own role. Admin can add, update and delete the tasks and attachments.
Developer Role
Developers have access rights to add, update and delete the tasks. Developer can also upload and modify or delete the attachments.
Contributor Role
Contributors can only update the tasks.
Project Members
Refers to those users who are actively working on the project. If the project creator is willing to work on the tasks as well, they'd need to add themselves as an active member, it' because a task can only be assigned to one of the project members. If the creator is added as a member their role will automatically be set to Creator. Project's creator role can not be changed. Clarification: since project creator already has the highest level of access to their project, adding them as a member is not to provide the access, it's only to be able to assign tasks to them.
Assignee
A task assignee must be one of the project's members.
External Collaborations
A user's "External Collaborations" refers to those projects which are not created by the user and user is just invited to work on those projects. So the user's role in external projects could be anything except Creator.
Active Tasks in context of profiles
Refers to those tasks which are not accepted.
Active Tasks in context of projects
Refers to those tasks which are not accepted and there's an assignee associated with the task who's working on it.
Available Tasks
Refers to those tasks which are not accepted and don't have an assignee.
